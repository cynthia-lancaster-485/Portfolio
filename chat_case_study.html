The stack

Every layer, designed and built from scratch.

Most designers hand off to developers. I am the developer. Building the entire stack myself meant every design decision was grounded in technical reality and every technical decision was informed by the user experience.

⚛️
Frontend
React + Tailwind

Component based UI with React Router for navigation and TanStack Query for data fetching.

🐍
Backend
Python / FastAPI

REST API with JWT authentication, protected routes, and full CRUD for accounts, chats, and messages.

🗄️
Database
PostgreSQL

Relational database storing users, chat rooms, and messages with proper foreign key relationships.

🔐
Auth
JWT Tokens

Secure login flow issuing Bearer tokens stored in localStorage. Protected routes redirect unauthenticated users.

What it does

A fully working chat platform.

Authentication

Register, login, stay logged in

Full registration and login flow with JWT tokens. Auth state persists across page refreshes via localStorage. Protected routes redirect unauthenticated users automatically.

Messaging

Send, edit, and delete messages

Real time message display with auto scroll to the latest message. Users can edit or delete their own messages inline. Other users' messages are read only.

Chat rooms

Multiple rooms, membership control

Browse a list of chat rooms from the sidebar. Membership is checked server side only members can send messages. Non members can read but not participate.

Account management

Update profile, password, delete account

Full account settings: update username and email, change password with current password verification, or permanently delete the account with a single confirmation.

Run it yourself

Want to run the full app locally?

The complete source code frontend, backend, and database setup is on GitHub. Follow the steps below to get everything running on your machine in about 5 minutes.

View on GitHub ↗
1
Prerequisites install these first
▼

Make sure you have these installed before you start. Click each link for the official installer.

🟢 Node.js

Version 20 or higher

nodejs.org ↗
🐍 Python

Version 3.11 or higher

python.org ↗
📦 Poetry

Python package manager

Install Poetry ↗
💡

To check if something is installed, open your terminal and run node --version, python --version, or poetry --version.

2
Clone the repository
▼

Download the code to your machine using Git. Open your terminal and run:

Terminal 
# Clone the repo
git clone https://github.com/cynthia-lancaster-485/Portfolio.git

# Navigate into the Chat Application folder
cd "Portfolio/Chat Application"
3
Set up and run the backend
▼

Install Python dependencies with Poetry, then start the FastAPI server. Run these from the project root (inside the Chat Application folder):

Terminal — Backend 
# Install all Python dependencies
poetry install

# Start the backend server
poetry run uvicorn backend:app --reload
✅

The backend will be running at http://127.0.0.1:8000. You can explore the auto-generated API docs at http://127.0.0.1:8000/docs — one of FastAPI's best features!

4
Set up and run the frontend
▼

Open a second terminal window (keep the backend running), navigate into the frontend folder, and start the Vite dev server:

Terminal — Frontend (new window) 
# Move into the frontend folder
cd frontend

# Install Node dependencies
npm install

# Start the frontend dev server
npm run dev
🚀

The app will be live at http://localhost:5173. Open that in your browser and you'll see the full Pony Express app running with a real backend and database!

5
Create an account and start chatting
▼

Once both servers are running, visit localhost:5173 in your browser. Click "Don't have an account? Register" to create a new account, then log in and start exploring. Open a second browser window with a different account to chat between two users in real time!

🧪

The API docs at 127.0.0.1:8000/docs ↗ let you explore and test every endpoint directly — great for seeing how the backend is structured.

What I learned

Building end to end changes how you design.

01

Auth is a UX problem

JWT tokens, protected routes, and persistent sessions aren't just backend concerns every decision affects what the user experiences. Designing the auth flow meant thinking about it from both sides simultaneously.

02

Owning the whole stack = faster iteration

When the designer and developer are the same person, there's no handoff delay, no misinterpretation, no "that's not possible." I could change the API contract and update the UI in the same commit.

03

State management is invisible UX

Getting auth context, message lists, and membership checks to stay in sync is invisible when it works and infuriating when it doesn't. Building it taught me to design with state in mind from the very start.
